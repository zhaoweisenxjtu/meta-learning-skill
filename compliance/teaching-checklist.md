# 教学执行清单

> ⚠️ 每次教学必须复制此清单到回复中，逐项完成并打勾。

## 讲解前检查
- [ ] 已确认用户当前层级（L1-L5）
- [ ] 已确认目标类型（考试型/应用型/兴趣型）
- [ ] 已调用 `meta-learn node content <nid>` 检查缓存
- [ ] 已调用 `meta-learn interaction list <uid> --node <nid>` 查看历史
- [ ] 已调用 `meta-learn misconception list --user <uid> --unresolved` 检查未纠正迷思

## 讲解执行（全部必做，按顺序）
- [ ] 1. 直觉构建：用类比/可视化/日常经验建立直觉（先于任何形式化定义）
- [ ] 2. 动机说明：回答"为什么需要这个概念"
- [ ] 3. 概念讲解：精确定义/公式 + 推导演化
- [ ] 4. 即时检验：每讲完一个概念立刻出 1 道检验题
- [ ] 5. 迷思预警：每个概念至少 1 个常见误解 + 纠正
- [ ] 6. 边界条件：明确适用范围和失效场景
- [ ] 7. 知识连接：从哪里来、和什么相连、通向哪里
- [ ] 8. 构造反例：至少 1 个"在什么情况下不成立"的问题
- [ ] 9. 信心校准：询问用户对自己答案的把握（1-5 分）

## 讲解后引擎记录（全部必做）
- [ ] `meta-learn interaction log` 记录本次教学
- [ ] `meta-learn review create` 记录 SM-2 评分
- [ ] `meta-learn quality assess` 做 NUSAP 评分
- [ ] `meta-learn misconception add` 记录发现的迷思（如有）
- [ ] `meta-learn weakness analyze` 分析薄弱模式

## 讲解后自评
- [ ] 完全不懂的人能建立直觉图像吗？
- [ ] 讲了"为什么"和"边界在哪里"吗？
- [ ] 指出了最常见的错误认知吗？
- [ ] 检验题能区分"真懂"和"假懂"吗？
- [ ] 知识点和其他知识的连接建立了吗？
