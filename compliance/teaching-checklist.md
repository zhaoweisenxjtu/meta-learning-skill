# 教学执行清单

> ⚠️ 每次教学必须复制此清单到回复中，逐项完成并打勾。
> 与主检查清单（SKILL.md）对齐，22 项全覆盖。

## 阶段一：入口判定
- [ ] 识别用户意图
- [ ] 模式判定完成 + 输出判定依据

## 阶段二：用户与路线
- [ ] 查询已有用户（meta-learn user list）
- [ ] 查询活跃路线（meta-learn track list <uid>）
- [ ] 输出当前状态（今日安排/待复习/进度）
- [ ] 用户确认路线选择

## 阶段三：诊断与评估（新路线首次必检）
- [ ] 概念自评（3-5个前置概念）
- [ ] 快速诊断测试（3-5道题，验证真实水平）
- [ ] 自评vs实测偏差判定（准确/高估/低估）
- [ ] 偏差处理（降级/鼓励/正常推进）

## 阶段四：教学执行（按顺序，不可跳过）
- [ ] 直觉构建：用类比/可视化/日常经验建立直觉（先于任何形式化定义）
- [ ] 动机说明：回答"为什么需要这个概念"
- [ ] 概念讲解：精确定义/公式 + 推导演化，每讲完一个概念立刻出 1 道检验题
- [ ] 迷思预警：每个概念至少 1 个常见误解 + 纠正
- [ ] 边界条件：明确适用范围和失效场景
- [ ] 知识连接：从哪里来、和什么相连、通向哪里
- [ ] 即时检验：至少 1 个检验问题 + 信心校准（1-5 分）+ 构造反例

## 阶段五：结构检验
- [ ] 分类任务：5 个相关概念和 5 个不相关概念混合分类
- [ ] 概念图：画概念关系图（连线标注关系类型）
- [ ] 费曼讲解：让用户用最简单语言解释给完全不懂的人（L5 检验）

## 阶段六：引擎记录（全部必做）
- [ ] `meta-learn interaction log` 记录本次教学
- [ ] `meta-learn review create` 记录 SM-2 评分
- [ ] `meta-learn quality assess` 做 NUSAP 评分
- [ ] `meta-learn misconception add` 记录发现的迷思（如有）
- [ ] `meta-learn weakness analyze` 分析薄弱模式

## 讲解后自评
- [ ] 完全不懂的人能建立直觉图像吗？
- [ ] 讲了"为什么"和"边界在哪里"吗？
- [ ] 指出了最常见的错误认知吗？
- [ ] 检验题能区分"真懂"和"假懂"吗？
- [ ] 知识点和其他知识的连接建立了吗？
