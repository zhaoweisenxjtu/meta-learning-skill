# 知识质量评估框架

基于知识图谱质量评估框架（工程界）和 NUSAP Pedigree Matrix，
为 meta-learning 知识库提供系统化的质量评估方法论。

## 一、评估框架总览

```
┌─────────────────────────────────────────────────────────┐
│                  知识质量评估框架                          │
├─────────────┬─────────────┬──────────────┬──────────────┤
│  模式层质量  │  数据层质量  │  NUSAP谱系   │  应用层质量   │
├─────────────┼─────────────┼──────────────┼──────────────┤
│ 节点类型    │ 准确性      │ 理论支撑     │ 检索召回率    │
│ 关系类型    │ 完整性      │ 数据来源     │ 答案忠实度    │
│ 本体设计    │ 一致性      │ 方法验证     │ 领域覆盖度    │
│ 分类体系    │ 时效性      │              │ 知识深度      │
│             │ 来源可信度   │              │              │
└─────────────┴─────────────┴──────────────┴──────────────┘
```

## 二、模式层质量

### 2.1 节点类型分类

每个知识节点必须归类到以下类型之一：

| 类型 | 代码 | 定义 | 示例 |
|------|------|------|------|
| 概念 | `concept` | 抽象概念或定义 | "机器学习"、"熵" |
| 事实 | `fact` | 可验证的事实性知识 | "地球绕太阳公转" |
| 原理 | `principle` | 科学原理或定律 | "牛顿第二定律" |
| 流程 | `procedure` | 步骤化方法或流程 | "SM-2算法步骤" |
| 框架 | `framework` | 结构化分析框架 | "PEST分析框架" |
| 案例 | `case` | 具体实例或案例 | "AlphaGo击败李世石" |
| 数据点 | `data_point` | 统计数据或数值 | "GPT-4参数1.8万亿" |
| 引用 | `reference` | 文献或来源引用 | "Roediger & Karpicke (2006)" |

### 2.2 关系类型规范化

节点间关系必须使用以下标准化类型：

| 关系类型 | 含义 | 示例 |
|---------|------|------|
| `prerequisite` | 前置知识 | 学习"微积分"前需先学"代数" |
| `related` | 相关概念 | "机器学习"与"统计学"相关 |
| `reference` | 引用/参考 | 某概念引用某论文 |
| `is_a` | 父子关系 | "决策树" is_a "监督学习" |
| `part_of` | 组成关系 | "注意力机制" part_of "Transformer" |
| `contradicts` | 矛盾/冲突 | "地心说" contradicts "日心说" |
| `supports` | 支持/佐证 | 实验数据 supports 某理论 |
| `applies_to` | 应用于 | "傅里叶变换" applies_to "信号处理" |

### 2.3 本体设计原则

1. **单一职责**：每个节点只描述一个知识点，不混入多个概念
2. **层次合理**：父节点是更抽象的概念，子节点是具体化
3. **关系明确**：关系类型必须准确反映节点间的实际联系
4. **无冗余**：同一知识点不应出现在多个节点中

## 三、数据层质量

### 3.1 准确性 (Accuracy)

知识内容是否真实反映客观事实。

**检查方法**：
- 与权威来源交叉验证
- 数值型知识核对原始数据
- 时间敏感型知识确认日期

**评分标准**：
- 4分：经多源交叉验证，与权威来源完全一致
- 3分：与主流知识一致，有可靠来源支撑
- 2分：基本正确，存在少量可接受的近似
- 1分：存在明显错误或过时信息
- 0分：内容错误或无法验证

### 3.2 完整性 (Completeness)

知识节点是否包含所有关键要素。

**理想知识节点应包含**：
1. 定义/描述 — 该知识是什么
2. 原理/机制 — 为什么是这样
3. 示例 — 具体应用场景
4. 边界条件 — 适用/不适用的情况
5. 关联知识 — 与其他知识的关系

**评分标准**：
- 4分：完整覆盖（定义+原理+示例+边界+关联）
- 3分：较完整（定义+原理+示例）
- 2分：基本完整（定义+简要说明）
- 1分：不完整（仅有名称或一句话）
- 0分：空节点

### 3.3 一致性 (Consistency)

知识节点内部及与其他节点之间是否逻辑自洽。

**检查维度**：
- **内部一致性**：节点内容不自相矛盾
- **外部一致性**：与同领域其他节点不冲突
- **跨域一致性**：与不同领域的相关知识一致

**评分标准**：
- 4分：与所有已知知识一致，无任何矛盾
- 3分：与主流知识一致，有少量未验证细节
- 2分：基本一致，存在可解释的局部矛盾
- 1分：存在明显矛盾或冲突
- 0分：自相矛盾

### 3.4 时效性 (Timeliness)

知识是否在有效期内。

**不同类型知识的默认有效期**：

| 节点类型 | 默认有效期 | 说明 |
|---------|-----------|------|
| 概念 | 5年 | 基础概念变化慢 |
| 事实 | 2年 | 事实性知识需定期更新 |
| 原理 | 10年 | 科学原理长期有效 |
| 流程 | 3年 | 方法流程可能更新 |
| 框架 | 5年 | 分析框架相对稳定 |
| 案例 | 1年 | 案例需保持时效 |
| 数据点 | 1年 | 统计数据快速变化 |
| 引用 | 10年 | 经典引用长期有效 |

### 3.5 来源可信度 (Source Reliability)

知识来源的权威性和可靠性。

**评分标准**：
- 4分：一手权威来源（官方统计、学术期刊、专利）
- 3分：高可信来源（行业白皮书、头部机构报告）
- 2分：中等可信来源（媒体报道、企业公开信息）
- 1分：低可信来源（论坛、自媒体、二手转述）
- 0分：不可追溯来源

## 四、NUSAP Pedigree Matrix

### 4.1 理论支撑等级 (Theory Level)

| 分数 | 等级 | 说明 |
|------|------|------|
| 4 | 成熟理论 | 公认定律、经典理论（如牛顿定律、热力学定律） |
| 3 | 广泛接受 | 领域内广泛接受的框架或模型（如SM-2算法、PEST分析） |
| 2 | 有论文支撑 | 有学术论文支撑但未成共识 |
| 1 | 经验推断 | 基于经验推断或类比 |
| 0 | 纯猜测 | 无理论依据 |

### 4.2 数据来源等级 (Data Level)

| 分数 | 等级 | 说明 |
|------|------|------|
| 4 | 官方/一手 | 官方统计、一手实测、权威数据库 |
| 3 | 行业报告 | 行业白皮书、头部机构报告（Gartner、McKinsey等） |
| 2 | 媒体报道 | 新闻报道、企业公开信息 |
| 1 | 二手转述 | 论坛、自媒体、二手转述 |
| 0 | 无来源 | 无法追溯来源 |

### 4.3 方法验证等级 (Method Level)

| 分数 | 等级 | 说明 |
|------|------|------|
| 4 | 独立验证 | 独立第三方验证/复现 |
| 3 | 同行评审 | 有同行评审（学术论文） |
| 2 | 内部验证 | 内部验证/逻辑自洽 |
| 1 | 有逻辑支撑 | 未验证但有逻辑推理 |
| 0 | 未验证 | 无任何验证 |

### 4.4 综合质量分计算

综合质量分 = NUSAP(40%) + 数据层质量(35%) + 时效性(25%)

```
overall = (theory + data + method) / 12 * 40    # NUSAP 40%
        + (completeness + consistency + source) / 12 * 35  # 数据层 35%
        + freshness_score                                    # 时效性 25%
```

时效性评分：
- fresh（>30天有效）：25分
- expiring_soon（≤30天有效）：15分
- stale（已过期）：5分

## 五、应用层质量

### 5.1 检索质量

评估知识库的检索效果：

| 指标 | 说明 | 测量方法 |
|------|------|---------|
| 召回率 | 相关节点被检索到的比例 | 测试查询集 |
| 精确率 | 检索结果中相关节点的比例 | 测试查询集 |
| F1分数 | 召回率和精确率的调和平均 | 综合计算 |

### 5.2 覆盖度

评估知识库对目标领域的覆盖情况：

| 指标 | 说明 |
|------|------|
| 领域覆盖率 | 目标领域知识点被覆盖的比例 |
| 节点类型均衡度 | 各类型节点的分布是否合理 |
| 深度分布 | L1-L5各层级节点的比例 |

**健康覆盖度参考**：
- 概念/原理/事实类节点应占主体（60%+）
- 案例/数据点作为补充（20-30%）
- 引用作为支撑（10-20%）
- L3+节点占比应≥60%

### 5.3 知识深度

评估知识库的深度质量：

| 层级 | 占比建议 | 说明 |
|------|---------|------|
| L1-L2（浅层） | ≤30% | 识别和复述层面 |
| L3（应用层） | 40-50% | 能正确应用 |
| L4（迁移层） | 15-25% | 能跨场景迁移 |
| L5（元认知） | 5-10% | 能教学和反思 |

## 六、评估流程

### 6.1 日常评估

每次复习或新增知识节点时自动触发：

1. 新节点创建 → 初始质量评估（`audit_type: initial`）
2. 复习完成 → 质量评分更新（`audit_type: review_update`）
3. 来源验证 → 来源可信度更新（`audit_type: source_verified`）

### 6.2 定期审计

| 频率 | 审计内容 | 触发方式 |
|------|---------|---------|
| 每周 | 时效性检查 | `audit_type: freshness_check` |
| 每月 | 交叉验证 | `audit_type: cross_validation` |
| 每季度 | 全面质量审计 | 手动触发 |

### 6.3 质量报告

使用 `meta-learn quality report <user_id>` 生成完整质量报告。

## 七、输出规范

### 7.1 节点质量标注

每个知识节点在输出时标注质量评分：

```
[概念] 机器学习
  质量评分：72/100
  NUSAP: (T:3, D:2, M:2)
  完整性：3/4 | 一致性：3/4 | 来源：2/4
  时效：fresh（剩余180天）
```

### 7.2 三元组标注

关键结论标注 (T,D,M) 三元组：
- (4,4,4) = 铁证：成熟理论 + 官方数据 + 独立验证
- (3,3,3) = 可靠：广泛接受 + 行业报告 + 同行评审
- (2,2,2) = 参考：有论文支撑 + 媒体报道 + 内部验证
- (1,1,1) = 推测：经验推断 + 二手来源 + 未验证
- (0,0,0) = 不可用：纯猜测 + 无来源 + 未验证

低于 (2,2,1) 的结论必须标注为推测并说明依据。
